# 🌟 演示案例系统使用指南

## 📖 功能介绍

演示案例系统允许您将**真实生成的作品**保存为演示案例，所有用户访问主页时都能看到您创作的精彩内容！

### 核心特点

✅ **真实生成** - 保存您真实创作的图片和视频
✅ **自动展示** - 在主页展示给所有用户
✅ **完整信息** - 包含剧本、分镜、提示词等完整信息
✅ **持续更新** - 可以继续生成后更新案例
✅ **永久保存** - 保存在数据库中，不会丢失

---

## 🚀 如何使用

### 1. 生成内容

首先，在项目工作台生成您的内容：

```
1. 输入剧本 → 生成分镜
2. 生成图片（通义万相）
3. 生成视频（豆包Seedance）
```

### 2. 保存为演示案例

生成完成后，点击页面右上角的 **"🌟 保存为演示"** 按钮：

```
┌────────────────────────────────────┐
│  💾 已保存 2026-3-15 23:45:12      │
│  🌟 保存为演示  🗑️ 清除进度   ⚙️   │
└────────────────────────────────────┘
```

### 3. 确认保存

系统会显示保存确认：

```
✅ 保存为演示案例成功！

案例ID: demo-a1b2c3d4
图片: 4张
视频: 2个

该案例将在主页展示给所有用户。
```

### 4. 查看效果

返回项目列表页，顶部会显示您的演示案例：

```
┌────────────────────────────────────┐
│     🌟 演示案例展示               │
│  查看其他用户创作的精彩案例        │
├────────────────────────────────────┤
│  [您的案例标题和图片预览]          │
│  📊 4 张图片  🎬 2 个视频          │
│  ⏱️ 20 秒                        │
└────────────────────────────────────┘
```

## 🔄 更新演示案例

如果您继续生成了更多内容，可以再次点击"🌟 保存为演示"按钮：

1. 系统会识别您已经保存过案例
2. 自动**更新**现有案例
3. 保留案例ID，添加新生成的内容
4. 所有用户立即看到最新版本

## 💡 使用建议

### 什么时候保存为演示案例？

**适合保存的情况：**
- ✅ 生成了高质量的内容
- ✅ 提示词优化得很好
- ✅ 视频效果令人满意
- ✅ 想要分享给其他用户
- ✅ 作为作品集展示

**不建议保存：**
- ❌ 测试用的内容
- ❌ 不完整的生成
- ❌ 质量不理想的内容

### 最佳实践

1. **优化提示词**
   ```
   普通提示：城市建筑
   优化提示：日出时分的城市天际线，金色的阳光洒满摩天大楼，朝霞满天
   ```

2. **完整生成**
   - 至少生成2-3张图片
   - 至少生成1-2个视频
   - 确保质量满意后再保存

3. **定期更新**
   - 生成新内容后及时更新
   - 保持案例的新鲜度
   - 展示最佳作品

## 🛠️ 技术细节

### 数据存储
- **后端**：SQLite数据库
- **表名**：demo_cases
- **持久化**：永久保存

### API接口

```
POST   /api/v1/demo-cases          # 创建演示案例
GET    /api/v1/demo-cases          # 获取案例列表
GET    /api/v1/demo-cases/{id}     # 获取单个案例
GET    /api/v1/demo-cases/latest   # 获取最新案例
PUT    /api/v1/demo-cases/{id}     # 更新案例
```

### 权限设置
- **公开**：默认公开，所有用户可见
- **只读**：其他用户只能查看，不能修改
- **不可删除**：只有管理员可以删除

## 📊 数据结构

### DemoCase 模型

```typescript
{
  id: string;              // 案例ID
  title: string;           // 标题
  description?: string;    // 描述
  script: string;          // 剧本

  updated_at: datetime;   // 更新时间
```

## ❓ 常见问题

**Q: 保存为演示案例会影响我的项目吗？**
A: 不会。演示案例是一个独立的副本，不影响您的原始项目。

**Q: 可以保存多个演示案例吗？**
A: 目前系统会更新同一个案例。如需多个案例，请创建不同项目。

**Q: 其他用户可以修改我的演示案例吗？**
A: 不能。演示案例是只读的，其他用户只能查看。

**Q: 如何删除演示案例？**
A: 目前需要联系管理员删除。未来会添加作者删除功能。

**Q: 演示案例会占用我的存储空间吗？**
A: 不会。演示案例保存在服务器数据库中，不占用您的浏览器存储。

**Q: 图片和视频链接会过期吗？**
A:
- 阿里云Dashscope的图片链接有效期约30天
- 豆包的视频链接有效期约24小时
- 过期后需要重新生成
